# ITSM – notions de base

## Définitions rapides
- **Incident** : quelque chose est cassé / ne marche plus (ex : VPN down).
- **Service request (Demande)** : l’utilisateur veut quelque chose de nouveau (ex : nouvel accès, nouveau PC).
- **Change** : modification planifiée et contrôlée d’un système (mise à jour, déploiement).

## SLA / Priorité
- **SLA** : engagement de délai (ex : first response < 2h, résolution < 8h).
- **Priorité = Impact × Urgence**
  - Impact = combien de personnes / business touchés
  - Urgence = à quelle vitesse il faut agir

## Exploitation
- **Escalade** : passage N1 → N2/N3 ou vers un manager si critique/complexe.
- **Knowledge Base (KB)** : articles de résolution / auto-dépannage.
- **Catégorisation + routing** : classer et assigner automatiquement au bon groupe.
